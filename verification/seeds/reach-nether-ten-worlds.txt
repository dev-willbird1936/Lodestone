# "reach-nether" fixed seed list for survival.reach-nether.
#
# No official multi-seed convention existed for this benchmark before this session (unlike B1's
# 20-seed, B2's 10-seed, and B3's 10-seed lists, which were all established in earlier sessions).
# This list mirrors B2/B3's own precedent directly: 10 arbitrary fixed Java world seeds, not
# pre-classified by spawn biome/terrain, because this repo has no way to know a seed's spawn
# biome, nearby water/lava, or ore availability without actually generating the 1.21.1 world. Each
# run's actual spawn conditions, gathered resources, and portal-build progress are read from that
# run's own worldObservation/goal-report telemetry after the fact, never assumed here.
#
# This is the hardest of the five benchmarks by a wide margin: the actor must gather or observe
# wood, stone, iron, gravel, water, AND lava (a strict superset of B3's stone-toolset
# requirements) before it can even attempt to assemble and light a portal, all through the same
# 3-segment checkpointed workflow (starter-tools / portal-tools / complete) that B2 and B3 use.
# Prior single-run ad-hoc realtime debugging sessions against this exact actor recorded 0 confirmed
# successes across 86 recorded attempts (see verification/evidence/minecraft-goal-reach-nether-
# realtime-*.json, most predating this session and run without a fixed seed or the current fix
# set) - that history is carried forward honestly here, not hidden: this benchmark's realistic
# pass bar is lower than B1-B3's, and a low or zero score on first official multi-seed rescoring
# would be consistent with its own prior evidence rather than a harness regression.
#
# Pass bar for this seed list: >=3/10 successful runs (reachedNether=true), each within the
# harness's 600000ms realtime / 480000ms script per-seed maxDurationMs cap. This bar is
# deliberately lower than B3's >=8/10 given the compounded multi-resource requirement and the 0/86
# prior informal success rate; if official scoring finds a seed failing for a genuine environment
# reason (e.g. no lava within the actor's bounded search radius), that seed may be swapped for
# another arbitrary one - any such swap must be called out explicitly in the benchmark report,
# never silently.
