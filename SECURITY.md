# Security policy

Lodestone defaults to loopback-only communication and default-deny mutation capabilities.

Report vulnerabilities privately through the repository's
[GitHub Security Advisory form](https://github.com/dev-willbird1936/Lodestone/security/advisories/new).
If that form is unavailable, open a non-sensitive issue asking for a private contact channel. Do not
include credentials, session tokens, private worlds, or player data in reports. Security-sensitive
changes require tests for authorization, bounded input, timeout, cancellation, and audit behavior.
